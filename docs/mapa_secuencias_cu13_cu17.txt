CU14 — NOTIFICAR VENCIMIENTO
═══════════════════════════════════════════════════════════

Actor: Proceso batch (ejecución MANUAL por consola)

⚠ ESTADO DE IMPLEMENTACIÓN: PARCIAL — NO AUTOMATIZADO
   • EXISTE un management command, pero hay que dispararlo a mano.
   • NO hay Celery, NO hay APScheduler, NO hay django-q, NO hay cron configurado en el proyecto.
   • NO hay signal de vencimiento. (licencias/signals.py solo maneja user_locked_out de django-axes.)
   • Para que sea realmente automático, falta un scheduler externo (cron del SO / Celery beat / etc.)
     que invoque el comando una vez al día.

FLUJO PRINCIPAL — ENVÍO DE ALERTAS (disparo manual)
──────────────────────────────────
1. Disparo: python manage.py enviar_alertas
      (NO hay URL ni vista; es un comando de consola)
2. Comando: Command.handle(self, *args, **kwargs)
      (archivo: licencias/management/commands/enviar_alertas.py, línea 11)
3. Cálculo de fecha objetivo:
      - hoy = timezone.now().date()                      (línea 12)
      - fecha_objetivo = hoy + timedelta(days=30)         (línea 15)
4. Lectura/ORM: Licencia.objects.filter(fecha_vencimiento=fecha_objetivo)
      (archivo: enviar_alertas.py, línea 19)
      - solo licencias que vencen EXACTAMENTE en 30 días
      - si no hay coincidencias → termina en silencio (líneas 22-24)
5. Armado del mensaje: itera licencias_a_vencer (líneas 29-39)
      - por cada licencia consulta lic.usuario_activo → empleado asignado (líneas 31-34)
6. Envío: send_mail(subject, message, from_email=settings.EMAIL_HOST_USER,
                    recipient_list=destinatarios, fail_silently=False)
      (archivo: enviar_alertas.py, línea 47)
      - destinatarios = ['arianyclaure@gmail.com']  (HARDCODEADO, línea 44)
7. Bitácora: NINGUNA (el comando NO registra en bitácora; solo escribe a stdout)
8. Response: salida por consola (SUCCESS / ERROR)

Infraestructura existente reutilizable para automatizar a futuro:
   • Campo Licencia.fecha_vencimiento (archivo: licencias/models.py, línea 190)
   • Propiedad Licencia.esta_vencida (usa fecha_vencimiento)
   • Licencia.cantidad_disponible (filtra fecha_vencimiento__gte=hoy)
   • Cálculo "por_vencer" (ventana 30 días) en gestionar_licencias() / dashboard

Actor: Administrador
Resumen: marca la(s) asignación(es) como inactivas (no borra) y libera la licencia.

FLUJO PRINCIPAL — LIBERAR A NIVEL LICENCIA
──────────────────────────────────
1. URL: POST /licencia/<int:licencia_id>/liberar/
      (archivo: config/urls.py, línea 68  → name='liberar_licencia')
2. Vista: liberar_licencia(request, licencia_id)
      (archivo: licencias/views.py, línea 642)
      - licencia = repo.get_licencia(licencia_id)
3. Use Case: uc_liberar_licencia(request=request, licencia=licencia) -> (ok, msg, cantidad)
      (archivo: licencias/application/use_cases/asignaciones.py, línea 67)
        a. asignaciones_activas = licencia.asignaciones.filter(activo=True).select_related('empleado')  (línea 69)
        b. if not exists() → (False, "...", 0)                                  (línea 70)
        c. captura nombres de empleados afectados                               (líneas 73-76)
        d. transaction.atomic(): por cada asignación:                           (línea 78)
              - asignacion.estado = 'LIBERADA'                                  (línea 80)
              - asignacion.activo = False                                       (línea 81)
              - asignacion.fecha_retiro = timezone.now() (si no tenía)          (líneas 82-83)

        e. licencia.estado_operativo = ESTADO_VENCIDA si esta_vencida, si no ESTADO_DISPONIBLE  (líneas 86-88)
        f. licencia.save(update_fields=['estado_operativo'])                    (línea 89)
4. Bitácora: log_liberar_licencia(request, licencia, empleados=[...], cantidad=count)
      (llamada en asignaciones.py línea 91; definición en bitacora/actions/licencias.py, línea 65)

      (archivo: config/urls.py, línea 71  → name='liberar_asignacion')

CU17 — GENERAR REPORTES (EXPORTAR A EXCEL)
═══════════════════════════════════════════════════════════

Actor: Administrador
Librería: openpyxl (3.1.5). NO usa csv, NO xlsxwriter, NO pandas.to_excel.

FLUJO PRINCIPAL — EXPORTAR REPORTE DE LICENCIAS
──────────────────────────────────
1. URL: GET /exportar/                       (todas las licencias)
      (archivo: config/urls.py, línea 59  → name='exportar_general')
   URL: GET /exportar/<int:tenant_id>/       (filtrado por tenant)
      (archivo: config/urls.py, línea 60  → name='exportar_tenant')
2. Vista: exportar_excel(request, tenant_id=None)
      (archivo: licencias/views.py, línea 70)
      - exigir_permiso(request, 'licencias.view_licencia')
3. Bitácora: log_exportar_excel(request, tenant_label=tenant_label)
      (archivo: bitacora/actions/licencias.py, línea 104)
      → log_event(accion=EXPORTAR, modulo=REPORTES, ...)
4. Construcción del libro:
      - wb = openpyxl.Workbook(); ws = wb.active; ws.title = "Reporte Completo"
      - cabeceras (17 columnas) + estilos (Font, PatternFill)
5. Query/ORM (lectura masiva con optimización N+1):
      - con tenant:  Licencia.objects.filter(tenant_id=tenant_id).select_related(
                        'tipo','empresa','tenant','proveedor','factura_origen')
      - sin tenant:  Licencia.objects.all().select_related(...)
6. Iteración: por cada licencia obtiene lic.usuario_activo (Asignacion activa) →
      empleado, email, centro de costos, área, división, unidad, fechas → ws.append([...])
7. Autoajuste de ancho de columnas (máx. 50)
8. Response (archivo descargable):
      - HttpResponse(content_type='application/vnd.openxmlformats-officedocument.spreadsheetml.sheet')
      - response['Content-Disposition'] = 'attachment; filename="Reporte_Licencias_<DD-MM-YYYY>.xlsx"'
      - wb.save(response); return response

Modelos leídos: Licencia (principal), Asignacion, Empleado, TipoLicencia, Tenant,
   Empresa, Proveedor, y estructura organizacional (Área/División/Unidad).
Escritura: ninguna (solo lectura).
Filtros: NO hay formulario HTML de filtros; el único filtro es tenant_id por URL
   (reporte completo "todo o nada" por tenant).
